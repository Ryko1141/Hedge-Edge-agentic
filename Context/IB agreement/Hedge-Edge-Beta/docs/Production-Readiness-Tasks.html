Hedge Edge App

Production Readiness — Task Specification Document
Date: February 10, 2026 Repo: github.com/Ryko1141/Hegde-app Payment Provider: Creem.io (MoR) Status: Pre-Production Audit
Context: Hedge Edge is an Electron + React desktop application for managing hedged trading across MT5, MT4, and cTrader platforms. The back-end is a FastAPI license validation API (Python) deployed at api.hedge-edge.com. This document lists all tasks required to bring the app from its current development state to production-ready, organized by priority. Creem.io is the chosen Merchant of Record for payments, subscription billing, and license key generation/validation.

CRITICAL Phase 1 — Must Complete Before Any Release

1. Integrate Creem.io for Payments & License Keys

⚠ Key Change: The current license system is entirely custom-built against Supabase tables and api.hedge-edge.com. There is zero Creem integration in the codebase today. This is the largest workstream (~2 weeks alone). Creem API endpoints: POST /v1/licenses/activate, POST /v1/licenses/validate, POST /v1/licenses/deactivate. Auth via x-api-key header. Test mode: https://test-api.creem.io, Production: https://api.creem.io.
#TaskDetails

1.7Build checkout/purchase flowIn-app purchase flow: open Creem checkout URL via shell.openExternal(). Handle return via deep link (hedgeedge:// protocol) or manual license key entry after purchase. Creem auto-generates keys on checkout.completed.
1.8Deploy webhook handlerEndpoint to receive Creem webhooks: checkout.completed (new license), subscription.canceled (revoke access), subscription.expired, subscription.past_due. Verify creem-signature header with HMAC-SHA256.

⚠ Biggest Workstream: The Creem.io integration (Tasks 1.1–1.10) is the single largest effort (~2 weeks). The current license system is entirely custom-built across 4 layers: React UI → Electron Main → Python API Proxy → MT5/MT4/cTrader EAs. All four layers must be updated to use Creem's license key activate/validate/deactivate endpoints. Creem handles license key generation automatically on checkout.completed — no need to generate keys yourself.
Creem.io API Quick Reference:
• Base URL (prod): https://api.creem.io  |  (test): https://test-api.creem.io
• Auth: x-api-key: creem_xxx header on all requests
• POST /v1/licenses/activate — body: { key, instance_name } → returns { id, status, key, activation, activation_limit, expires_at, instance }
• POST /v1/licenses/validate — body: { key, instance_id } → returns same schema, check status = "active"
• POST /v1/licenses/deactivate — body: { key, instance_id } → frees activation slot
• Webhook events: checkout.completed, subscription.active, subscription.paid, subscription.canceled, subscription.expired
• Webhook signature: HMAC-SHA256 of request body, sent in creem-signature header
• Pricing: 3.9% + 40¢ per transaction (flat, no hidden fees). MoR handles tax compliance globally.
